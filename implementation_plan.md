# 📋 تحليل متطلبات مشروع مغسلتي بلس (MyLaundryPlus)

## نظرة عامة على المشروع

**مغسلتي بلس** هو نظام SaaS متكامل لإدارة المغاسل في دول الخليج واليمن، يتكون من ثلاثة تطبيقات متكاملة تعمل معاً عبر واجهة برمجية موحدة (Unified API). اللون الرئيسي للمشروع: **أزرق** مع تصميم عصري.

---

## 🏗️ البنية التقنية المقترحة (Tech Stack)

| الطبقة | التقنية | السبب |
|--------|---------|-------|
| **Admin Dashboard** | React.js + Vite | سرعة التطوير، مكونات إعادة استخدام |
| **Customer App** | React Native (Expo) | iOS + Android من كودبيس واحد |
| **Laundry Panel** | React Native أو React.js | يعمل على تابلت ومحطة POS |
| **Backend API** | Node.js + Express.js أو Python (FastAPI) | الأخير مناسب للتحليلات |
| **قاعدة البيانات** | PostgreSQL + Redis (Cache) | علائقية قوية + كاش للأداء |
| **الإشعارات** | Firebase Cloud Messaging (FCM) | مجاني وموثوق |
| **التحليلات** | Python (FastAPI + Pandas) | كما طلب العميل |
| **OTP / SMS** | Twilio أو Unifonic (للخليج) | دعم أرقام السعودية والخليج |
| **الخرائط** | Google Maps API | دعم قوي لمنطقة الخليج |
| **التخزين** | AWS S3 أو Cloudflare R2 | صور، فواتير PDF |
| **الطباعة** | Bluetooth Thermal Printer SDK | طابعات POS الحرارية |

## 1️⃣ نظام الإدارة — Admin Dashboard (React.js)

### 📌 الوحدات والمتطلبات التفصيلية

#### أ) لوحة المؤشرات (Analytics Dashboard)
- **KPIs رئيسية**: إجمالي الإيرادات، عدد المغاسل النشطة، عدد العملاء، عدد الطلبات اليومية
- **رسوم بيانية**: إيرادات الاشتراكات (خطي)، المغاسل الأكثر نشاطاً (شريطي)، حركة العملاء (يومي/أسبوعي/شهري)
- **تقارير قابلة للتصدير**: PDF وExcel

#### ب) إدارة الاشتراكات (Subscription Management)
```
خطط الاشتراك:
- شهري / ربعي / سنوي
- تجريبي (Trial Period - قابل للتخصيص)
- فترة النعمة (Grace Period) بعد انتهاء الاشتراك

العمليات:
✅ إضافة خطة اشتراك جديدة
✅ تعديل السعر والمميزات
✅ حذف خطة
✅ إضافة كود خصم (Promo Code) - نسبة أو مبلغ ثابت
✅ تحديد صلاحية كود الخصم (تاريخ انتهاء، عدد مرات الاستخدام)
✅ تتبع المغاسل المنتهية اشتراكاتها (تنبيه تلقائي)
```

#### ج) إدارة الإعلانات (Ads Management)
```
أنواع الإعلانات:
- إعلانات للعملاء (عروض عامة)
- إعلانات للمغاسل (معدات غسيل، مستلزمات)

مكونات الإعلان:
- العنوان (Title)
- الوسائط (Media) — دعم رفع أكثر من صورة + فيديو في نفس الإعلان
- النص التفصيلي (Description)
- تاريخ البداية والنهاية
- الجمهور المستهدف (عملاء / مغاسل / الكل)
- الحالة (نشط / موقوف)

العمليات:
✅ إضافة / تعديل / حذف إعلان
✅ رفع متعدد للوسائط (صور + فيديو) مع إمكانية إعادة الترتيب
✅ معاينة الإعلان قبل النشر
✅ إحصاء عدد المشاهدات
```

#### د) إدارة المستخدمين (User & Vendor Management)
```
إدارة العملاء:
✅ عرض قائمة العملاء مع البحث والفلترة
✅ عرض تفاصيل الحساب (الاسم، الرقم، المعرف الفريد، QR Code)
✅ تفعيل / تعطيل الحساب
✅ إدارة الأجهزة المرتبطة بالحساب:
   - عرض جميع الأجهزة النشطة (نوع الجهاز، OS، آخر دخول)
   - تفعيل جهاز / إلغاء تفعيله بشكل منفرد
   - تحديد جهاز افتراضي (Primary Device)
   - دعم أكثر من جهاز في نفس الوقت (Multi-Device)
✅ حذف الحساب
✅ تعديل البيانات

إدارة المغاسل:
✅ عرض قائمة المغاسل مع الحالة (نشط/تجريبي/منتهي/محظور)
✅ تفعيل المغسلة بعد الاشتراك أو انتهاء الفترة التجريبية
✅ حظر المغسلة المخالفة
✅ عرض تفاصيل المغسلة (البيانات، الاشتراك، الإيرادات)
✅ إدارة أجهزة المغسلة (نفس آلية إدارة أجهزة العميل)
✅ إضافة مغسلة يدوياً
✅ تعديل بيانات المغسلة
```

#### هـ) الدعم الفني والشكاوى (Tickets / CRM System)
```
قنوات الاستقبال:
- رسائل مباشرة (Chat) من تطبيق العميل إلى الإدارة
- رسائل مباشرة (Chat) من تطبيق المغسلة إلى الإدارة
- مكالمات هاتفية واردة من العملاء أو المغاسل
- مكالمات صادرة من الإدارة نحو العملاء أو المغاسل (مثل نظام CRM لخدمة العملاء)

إدارة التذاكر:
✅ عرض قائمة التذاكر (مفتوحة / قيد المعالجة / مغلقة)
✅ الرد على التذكرة عبر Chat مباشر
✅ تسجيل ملاحظات المكالمة الهاتفية يدوياً على التذكرة
✅ إحالة التذكرة لموظف آخر
✅ إغلاق التذكرة وتقييم الحل
✅ تعديل أرقام التواصل المعروضة في التطبيقات (تُحدَّث من لوحة الإدارة وتنعكس فوراً)
```

## 2️⃣ تطبيق العميل — Customer App (React Native)

### 📌 الوحدات والمتطلبات التفصيلية

#### أ) إدارة الحساب (Account Management)
```
التسجيل والدخول:
- التسجيل: الاسم + رقم الهاتف
- اختيار البلد عند التسجيل (يحدد العملة الافتراضية تلقائياً)
- التحقق عبر OTP (SMS)
- توليد معرف فريد تلقائي (UUID مختصر + QR Code)
- إضافة صورة شخصية (اختياري)
- تسجيل نوع الجهاز تلقائياً عند الدخول (إلزامي لإدارة الأجهزة)
- الموقع الجغرافي (اختياري - يُستخدم للخريطة فقط)

إدارة الأجهزة (Multi-Device):
✅ دعم تسجيل الدخول من أكثر من جهاز
✅ عند الدخول من جهاز جديد: عرض نوع الجهاز وطلب التأكيد
✅ قائمة بجميع الأجهزة النشطة (النوع، OS، تاريخ آخر دخول)
✅ تفعيل / إلغاء تفعيل أي جهاز
✅ تحديد جهاز افتراضي (Primary Device)

بيانات الملف الشخصي:
✅ الاسم، رقم الهاتف
✅ المعرف الفريد + QR Code (قابل للمشاركة)
✅ البلد والعملة المفضلة (قابلة للتعديل في الإعدادات)
✅ تاريخ الانضمام
✅ إجمالي الفواتير والمبالغ
```

#### ب) الاستكشاف والفلترة (Discovery & Filtering)
```
خريطة تفاعلية:
✅ عرض المغاسل القريبة على خريطة Google Maps
✅ تحديد الموقع الحالي للعميل (اختياري - يطلب الإذن)
✅ إمكانية تصفح المغاسل دون مشاركة الموقع (يدوياً)
✅ عرض تفاصيل المغسلة عند الضغط على الدبوس

خيارات الفلترة:
✅ الأعلى تقييماً
✅ الأقرب جغرافياً
✅ الأرخص سعراً
✅ المفتوح الآن (بناءً على أوقات العمل)
```

#### ج) تتبع الطلبات (Order Tracking)
```
حالات الفاتورة (State Machine):
🔵 تم الاستلام → قيد الغسيل → قيد الكوي → جاهز للاستلام → مكتمل

ملاحظات تقنية مهمة:
- كل تغيير في الحالة يُرسل Push Notification للعميل
- لا يمكن العودة لحالة سابقة (للحفاظ على سلامة البيانات)
- الوقت المقدر لكل مرحلة (اختياري)
```

#### د) العروض والإعلانات (Promotions Screen)
```
✅ شاشة مخصصة لعرض إعلانات من الإدارة
✅ عروض المغاسل المحلية
✅ فلترة العروض حسب المغسلة أو النوع
✅ روابط قابلة للنقر (لتفاصيل أكثر)
```

#### هـ) التواصل والتقييم (Communication & Rating)
```
المحادثة النصية (In-App Chat):
✅ دردشة مباشرة مع المغسلة
✅ دردشة مع الدعم الفني (الإدارة الرئيسية للنظام)
✅ إرسال صور (توثيق حالة الملابس)
✅ إشعارات بالرسائل الجديدة

التواصل مع الدعم:
✅ زر "تواصل معنا" في الإعدادات يعرض أرقام هاتفية (مُضافة من Dashboard)
✅ زر محادثة نصية مباشرة مع فريق الدعم
✅ الأرقام الظاهرة تُدار وتُعدَّل من Admin Dashboard

نظام التقييم الذكي:
✅ يُفتح فقط عند حالة "مكتمل"
✅ تقييم من 1-5 نجوم
✅ تعليق اختياري
✅ منع إعادة التقييم لنفس الفاتورة
```

#### و) سجل الفواتير (Invoice History)
```
✅ قائمة جميع الفواتير (مع الفلترة بالتاريخ والحالة)
✅ تفاصيل كل فاتورة (الأصناف، الكميات، الأسعار، الإجمالي)
✅ حالة الفاتورة الحالية
✅ المبالغ الآجلة المتأخرة (إن وجدت)
✅ زر مشاركة الفاتورة
```

## 3️⃣ تطبيق/لوحة المغسلة — Laundry Panel (POS)

> **ملاحظة المنصة**: الأساس تطبيق هاتف (React Native)، مع دعم كامل لتابلت/iPad عند الحاجة. التصميم يكون Responsive ليتكيف مع حجم الشاشة.

### 📌 الوحدات والمتطلبات التفصيلية

#### أ) شاشة تسجيل الدخول / إنشاء الحساب (Auth Screen)
```
إنشاء حساب جديد:
✅ الاسم التجاري للمغسلة + رقم الهاتف
✅ التحقق عبر OTP
✅ تسجيل نوع الجهاز إلزامياً (للتحكم في تفعيل الأجهزة لاحقاً)
✅ الموقع الجغرافي (اختياري عبر GPS أو يدوي)

تسجيل الدخول من جهاز جديد:
✅ عرض نوع الجهاز الجديد وطلب التأكيد
✅ إشعار لصاحب الحساب بمحاولة الدخول الجديدة

إدارة الأجهزة (Multi-Device):
✅ دعم أكثر من جهاز (هاتف + تابلت مثلاً)
✅ قائمة بجميع الأجهزة النشطة
✅ تفعيل / إلغاء تفعيل أي جهاز
✅ تحديد جهاز افتراضي
```

#### ب) إدارة حساب المغسلة (Laundry Profile)
```
البيانات الأساسية:
✅ اسم المحل
✅ رقم الإدارة الداخلي
✅ رقم التواصل للعملاء (يظهر في تطبيق العميل)
✅ الموقع الجغرافي الدقيق (GPS أو إدخال يدوي)
✅ أوقات العمل (افتتاح/إغلاق لكل يوم في الأسبوع)
✅ صورة الشعار/البروفيل (اختياري)
✅ حالة المحل (مفتوح/مغلق) - تبديل يدوي سريع
```

#### ب) دليل الأصناف (Menu / Catalog Management)
```
هيكل القائمة:
القسم (Category) → الصنف (Item) → السعر (Price)

مثال:
- رجالي → ثوب كوي → 3 ريال
- رجالي → بنطال → 2 ريال
- نسائي → عباءة → 5 ريال

العمليات:
✅ إضافة / تعديل / حذف قسم
✅ إضافة / تعديل / حذف صنف داخل قسم
✅ دعم الوحدات (قطعة، كيلو)
✅ إخفاء صنف مؤقتاً (دون حذفه)
```

#### ج) نظام الفواتير (Invoicing - POS)
```
إنشاء الفاتورة:
✅ البحث عن العميل بالمعرف الفريد أو الاسم
✅ مسح QR Code للعميل (كاميرا الجهاز)
✅ إضافة أصناف من القائمة (مع الكمية)
✅ حساب الإجمالي تلقائياً
✅ إضافة ملاحظات على الفاتورة

أنواع الدفع:
💵 نقدي (Cash)
💳 شبكة (POS Machine)
📋 آجل (Deferred - للعملاء الموثوقين)

الطباعة والمشاركة:
✅ طباعة عبر طابعة بلوتوث حرارية (ESC/POS Protocol)
✅ طباعة عبر WIFI
✅ تصدير PDF وإرسال عبر واتساب
✅ إرسال إشعار تلقائي للعميل في التطبيق

إدارة حالات الفاتورة:
تم الاستلام ← قيد الغسيل ← قيد الكوي ← جاهز ← مكتمل
(كل تغيير يُرسل Push Notification للعميل)
```

#### د) إدارة العملاء - CRM
```
✅ سجل كامل بعملاء المغسلة
✅ عرض تاريخ كل عميل مع المغسلة (فواتيره فقط)
✅ تتبع مبالغ الآجل المتأخرة
✅ إرسال تنبيه للعميل عند تعديل فاتورته
✅ تصفية العملاء حسب المبلغ المتأخر
```

#### هـ) الترويج (Promotions)
```
✅ إنشاء عرض خاص بالمغسلة
✅ اختيار الجمهور (عملاء المغسلة أو عام)
✅ تحديد مدة العرض
✅ إرفاق صورة للعرض
```

#### و) التقارير اليومية (Daily Reports)
```
ملخص يومي:
✅ إجمالي الدخل اليومي
✅ توزيع: نقدي / شبكة / آجل
✅ عدد الفواتير
✅ أكثر الأصناف طلباً
✅ تصدير التقرير PDF
```

## 🔧 المتطلبات التقنية المشتركة (Cross-Cutting Concerns)

### نظام الإشعارات (Push Notifications - FCM)
| الحدث | المستقبل |
|-------|---------|
| إنشاء فاتورة جديدة | العميل |
| تغيير حالة الفاتورة | العميل |
| فاتورة جاهزة للاستلام | العميل |
| تعديل على فاتورة آجلة | العميل |
| انتهاء اشتراك المغسلة | صاحب المغسلة |
| رسالة دعم فني جديدة | الإدارة |

### نظام الحالات (Invoice State Machine)
```
[جديدة] → [تم الاستلام] → [قيد الغسيل] → [قيد الكوي] → [جاهز للاستلام] → [مكتمل]
                                                                              ↑
                                                                    (يُفتح التقييم هنا فقط)
```
> ⚠️ لا يُسمح بالتراجع عن الحالات لمنع التلاعب في الحسابات

### المصادقة والأمان (Auth & Security)
```
- OTP عبر SMS لتطبيق العميل والمغسلة (Twilio / Unifonic)
- JWT Tokens مع Refresh Token
- دعم Multi-Device: تسجيل دخول من أكثر من جهاز في نفس الوقت
- تسجيل نوع الجهاز إلزامياً عند أي دخول جديد
- إدارة الأجهزة من Admin Dashboard (تفعيل / إلغاء / تحديد افتراضي)
- تشفير البيانات الحساسة (HTTPS، bcrypt للكلمات السرية)
```

### التحليلات (Analytics - Python)
```
خدمة Python منفصلة (FastAPI):
- تحليل بيانات المغاسل (أكثر الأوقات نشاطاً)
- تقارير الإيرادات للإدارة
- الكشف عن الحسابات المشبوهة
- توقعات الطلب الموسمي
```
